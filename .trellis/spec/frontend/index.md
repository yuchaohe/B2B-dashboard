# Frontend Development Guidelines

> Minimum frontend conventions for the week-1 admin dashboard.

---

## Overview

This package defines the initial frontend rules for a React + TypeScript internal admin dashboard.

Week 1 focuses on:

- app shell and routing
- overview page
- customer and order route placeholders
- API-driven mock data rendering

## Week-1 Review Checklist

- Is the new UI work placed under a clear route or feature boundary?
- Are page components thin and data access delegated to API modules?
- Are shared layout elements separated from feature-specific components?
- Are types declared or imported instead of inferred from ad hoc object literals everywhere?
